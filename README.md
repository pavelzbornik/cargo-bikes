# cargo-bikes — Public Obsidian Vault for Cargo Bikes

This repository is a public Obsidian-style vault that stores structured information about cargo bikes, with a focus on long-tail (longtail) cargo bikes. Use this vault as a knowledge base for models, design notes, specifications, maintenance tips, and community contributions.

## Contents

- `/.obsidian/` — notes about recommended Obsidian setup for this vault.
- `/vault/notes/` — the main notes; each bike, article or topic is a separate Markdown file organized by brand.
- `/vault/templates/` — templates for adding new bike entries, specs, and general notes.
- `/scripts/` — Python utilities for generating the bike table and validating content.
- `/tests/` — pytest test suite for Python scripts.
- `/docs/` — documentation for Python utilities.
- `CONTRIBUTING.md` — contribution rules, metadata/frontmatter conventions and PR checklist.

## Why this repo

Long-tail cargo bikes are a popular and practical platform for carrying kids, cargo, and commuting at scale. This vault is intended to collect manufacturer specs, rider notes, mod lists, maintenance tips, fit guidelines, and comparative analyses in a discoverable, linked-note format.

## Quick start

1. Clone the repo and open it as a vault in Obsidian or any Markdown editor.
2. Add new notes under `vault/notes/` using the templates in `vault/templates/`.
3. Follow the frontmatter conventions in `vault/README.md` and `CONTRIBUTING.md`.

**Frontmatter example:**

```yaml
---
title: "Yuba Mundo (example)"
type: bike
brand: "Yuba"
model: "Mundo"
tags: ["long-tail", "family", "electric-compat"]
date: 2025-10-16
---
```

## Python Utilities and Automation

### Environment Setup

Python 3.10+ is required for running scripts and tests. Dependencies are managed via `pyproject.toml`:

1. **Install uv** (Python package manager): [https://github.com/astral-sh/uv](https://github.com/astral-sh/uv)
2. **Create and sync virtual environment:**

   ```sh
   uv sync
   ```

3. **Activate the venv:**
   - Windows PowerShell: `.venv\Scripts\Activate.ps1`
   - Linux/Mac: `source .venv/bin/activate`

### Bike Table Generator

The `scripts/generate_bike_table.py` script automatically generates the bike table in `README.md` by reading YAML frontmatter from all bike notes.

**Usage:**

```sh
python scripts/generate_bike_table.py
```

**Key features:**

- Scans `vault/notes/*/*.md` for bike entries
- Validates required frontmatter fields: `title`, `type: bike`, `tags`
- Generates a sorted Markdown table with specs and images

**When to run:**

- After adding or modifying bike notes
- Before committing changes to ensure README is up to date

See `docs/GENERATE_BIKE_TABLE.md` for complete documentation.

### URL Validator

**Script:** `scripts/validate_urls.py`

**Purpose:** Validates URL links in bike note frontmatter fields (`url` and `image`). Checks if URLs are reachable via HTTP requests, removes invalid URLs from the frontmatter, and logs all changes in a JSON changelog.

**Usage:**

```sh
python scripts/validate_urls.py                 # Validate all URLs
python scripts/validate_urls.py --dry-run       # Preview changes without modifying
python scripts/validate_urls.py --skip-check    # Only check URL structure (no HTTP requests)
```

**Output:**

- Invalid URLs are removed from frontmatter
- Changes are logged to `VALIDATE_URLS_CHANGELOG.json`
- Console output shows real-time progress and summary statistics

**When to run:**

- Periodically to check for broken links in bike notes
- After batch updates to bike notes
- Before major releases

See `docs/VALIDATE_URLS.md` for complete documentation.

### Testing

Tests are run with pytest:

```sh
pytest                      # Run all tests
pytest --cov=scripts        # Run with coverage
```

Tests validate:

- YAML frontmatter extraction and parsing
- Bike frontmatter field validation
- Table generation logic
- README.md update functionality

## Linting and formatting Markdown (pre-commit)

To ensure all Markdown files are linted and formatted before commit, this repo uses [pre-commit](https://pre-commit.com/) with `markdownlint` and `prettier` hooks.

**Setup:**

1. Activate the Python venv (see "Python Utilities and Automation" above)
2. Install Node.js (required for markdownlint and prettier): [https://nodejs.org/](https://nodejs.org/)
3. Install the hooks:

```sh
pre-commit install
```

**Usage:**

- Hooks will run automatically on `git commit`.
- To manually check all files:

```sh
pre-commit run --all-files
```

**Configuration:**

- See `.pre-commit-config.yaml` for details.
